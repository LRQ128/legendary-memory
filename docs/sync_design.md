# L&W 账本 - 登录+数据同步方案设计

> 设计日期：2026-06-23
> 目标：让用户登录后数据自动同步到云端，换手机登录数据不丢失

---

## 一、技术选型：LeanCloud

经过综合评估，推荐使用 **LeanCloud（leancloud.cn）**：

| 项 | 说明 |
|:---|:------|
| 🏢 **提供商** | 美国公司在中国运营，有国内合规资质 |
| 🇨🇳 **国内可用** | ✅ 有 `.cn` 域名，华为手机无问题 |
| 📱 **Flutter SDK** | ✅ `leancloud_storage` 官方 SDK |
| 💰 **价格** | 免费额度：100万次API/月、1GB存储，足够个人记账 |
| 🔐 **数据安全** | 用户数据隔离（每个用户仅看自己数据） |
| ⚡ **延迟** | 国内服务器，响应快 |

### 为什么不用别的？

| 方案 | 排除原因 |
|:---|:---------|
| Firebase | ❌ 华为手机无 Google Play Services |
| Supabase | ❌ 国外服务器国内慢，无合规背书 |
| 自建后端 | ❌ 需要服务器+维护，成本高 |
| 华为AGC | ⚠️ 可做备选，但开发者注册门槛较高 |

## 二、整体架构

```
┌──────────────────────────────────────┐
│           手机端 App                   │
│                                        │
│  ┌──────────┐    ┌───────────────┐    │
│  │ SQLite    │ ←→│ SyncService   │    │
│  │ (本地DB)  │    │ (同步引擎)     │    │
│  └──────────┘    └──────┬────────┘    │
│       ↑                 │             │
│       │ 离线也可用       │ 有网时同步   │
│       └─────────────────┘             │
└──────────────────────────┼───────────┘
                           │ HTTPS REST
                    ┌──────▼───────┐
                    │  LeanCloud   │
                    │   云存储      │
                    │  ┌────────┐  │
                    │  │ User A │  │
                    │  │ User B │  │
                    │  └────────┘  │
                    └──────────────┘
```

**核心原则：** 离线优先（Offline-first）

- 所有数据先存本地 SQLite
- 有网络时后台同步到云端
- 无网络时正常使用，有网后自动同步
- 用户完全无感知

## 三、数据模型设计

### Transaction 表（改动）

```
字段         类型        说明                         同步相关
─────────────────────────────────────────────────────────────
id          int         SQLite自增主键
remoteId    String?     LeanCloud的objectId（关联云端）
type        String      'income' / 'expense'
amount      double      金额
category    String      分类
note        String      备注
date        DateTime    日期
account     String      账户
createdAt   DateTime    创建时间
images      String      图片路径（|分隔）
updatedAt   DateTime    ★ 最后修改时间（同步依据）
syncStatus  int         ★ 同步状态：0=未同步, 1=已同步, 2=待上传
```

### SyncService 工作流程

```
用户操作 (增/删/改)
    ↓
1. 写入本地 SQLite（标记 syncStatus=2 待上传）
    ↓
2. SyncService 检查：
   ├─ 无网络 → 跳过，下次有空再同步
   └─ 有网络 → 批量上传变更到 LeanCloud
         ↓
3. 上传成功 → 更新 syncStatus=1, 记录 remoteId
         ↓
4. 拉取云端其他设备的最新数据
         ↓
5. 对比 updatedAt：
   ├─ 云端更新 → 下载覆盖本地
   └─ 本地更新 → 已上传，跳过
```

## 四、登录流程

```
首次使用：
┌──────────┐     ┌────────────┐     ┌────────────────┐
│ 注册页面  │ ──→ │ 手机号+密码 │ ──→ │  LeanCloud     │
│ 或登录页  │     │ (或邮箱)    │     │  创建账户       │
└──────────┘     └────────────┘     └───────┬────────┘
                                            ↓
                                    判断是否首次登录
                                    ├─ 是 → 无云端数据
                                    │         ↓
                                    │    本地数据 → 上传到云端
                                    │
                                    └─ 否 → 有云端数据
                                              ↓
                                         云端数据 → 下载到本地
                                         与本地数据合并（取最新）
```

**登录方式：** 手机号/邮箱 + 密码（LeanCloud 内置支持）

**记住登录：** 使用 `flutter_secure_storage` 保存 token，下次自动登录

## 五、数据同步规则（关键）

### 合并策略：last-write-wins（LWW）

个人记账软件**不会有多人同时修改同一条数据**的场景，所以：
- **每条记录有 `updatedAt` 时间戳**
- **同步时谁的时间戳新就用谁的**
- 无需复杂的冲突解决（CRDT 等）

### 首次登录合并

```
本地已有数据 + 云端已有数据（换手机场景）
    ↓
1. 下载云端所有数据到内存
2. 对每条记录比较 updatedAt：
   ├─ 仅本地有 → 上传到云端
   ├─ 仅云端有 → 下载到本地
   └─ 两端都有 → 取 updatedAt 最新的版本
3. 完成后本地和云端完全一致
```

### 图片同步策略

| 方式 | 说明 |
|:---|:------|
| **不直接同步图片文件** | 图片体积大，上传慢 |
| **云端只存路径索引** | LeanCloud 上存图片文件名列表 |
| **图片用云存储** | 可选：用户自行备份，或集成 LeanCloud File |

**简化方案（推荐）：** 先不同步图片文件，只同步数据文本。图片后续可加。
